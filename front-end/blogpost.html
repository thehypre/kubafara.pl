About the site

This website was created by me to apply the skills I learned while getting an AWS Cloud Practitioner certification. The goal was to create a CV webpage with the simple functionality of a view counter implemented in a not-so-simple AWS environment.

The first step was setting up a static page using an S3 bucket and a CloudFront distribution. The page redirects all HTTP traffic to HTTPS. Using an external domain provider, I purchased a .pl domain and set it up with CloudFront using CNAME records added to the home.pl admin panel. The website is based on a modified custom template from templateflip.com

In the second step, I manually implemented a simple JavaScript function on the frontend side and a Python Lambda Function on the backend to get a value from a DynamoDB database via Function URL and increment it by one.

The final step was to set up most of the things again, this time using IaaC with a CI/CD pipeline. I chose Terraform since it is not an Amazon service, but a widely adapted tool used by other cloud providers. The CI/CD pipeline made in GitHub Actions provides a sufficient level of automation - for the frontend, it updates the webpage contents in the S3 bucket and invalidates the CloudFront, so any changes are to be seen by a user in a matter of seconds. The backend part handles and applies the terraform changes to the infrastructure while ensuring the quality of the code.

Creation of the website was inspired by the Cloud Resume Challenge. Working on this simple project showed me how much there is yet to learn in the cloud for me. Even a simple view counter can be challenging for a person new to AWS Services; however, this only made me more eager to learn more. Also in this project, I could still try to do things differently: try to build CloudFront in the Terraform even with an external provider, create a new bucket to store Terraform state more securely, try to use API Gateway instead of Function URL, include tests for the backend and frontend, etc. For the next challenge, I would like to build a proper web app and include different topics like Docker, Kubernetes, Ansible, advanced networking and security in the cloud.
